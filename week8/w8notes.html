Chapter 8 Summary - Transforms and Transitions

  • The "transform" property lets you translate, rotate, scale,
    and/or skew any elements on the page.
  • We can manipulate an elements appearance using transform functions.
  • Translate Functions allow you to move elements left, right, up, & down
    - similar to "position: relative" but using translate allows you to
    move elements without impacting the flow of the document.
    • translate(x,y) function moves an element x from the LEFT,
      and y from the TOP.
    • Ex: .ad h1:hover span { transform: translate(45px, -45px); }
    • For only up or down use: transform: translateX(45px); }
      -OR- transform: translateY(-45px); }
      * Note: Need to add "display: inline-block;" to the span for this to work.
  • scale(x,y) function: scales by the defined factors horizontally then vertically.
    • If only one value is provided it will be used for both x and y.
    • It grows and shrinks from its center.
    • scale(1) would keep the image the same.
    • scale(2) would double it.
    • scale(0.5) would halve it.
  • rotate() function: rotates around the point of origin.
    • rotate(90deg) turns the image to face right.
    • rotate(180deg) turns the image upsidedown.
    • rotate(-90deg) turns it to face left.
    • *Note: positive numbers turn it clockwise, negative turns it counter-clockwise.
  • skew(x,y) function: specifies a skew along the x and y axes.
    If a second parameter is omitted, the skew will only occur on the x axis.
  • Transitions: they allow the values of CSS properties to change over time,
    providing simple animations.
    • Transitions are declared along with the regular styles on an element.
    • Steps to create a simple transition using only CSS:
      1. Declare the original state of the element in the default style declaration.
      2. Decalre the final state of your transitioned element; for example a :hover state.
      3. Include the transition functions in your default style declaration
        using the transition properties - including: transition-property,
        transition-duration, transition-timing-function, & transition-delay.
  • Animations: unlike transitions, CSS animations allow you to control each step of
    an animation via keyframes.
    • A keyframe is a snapshot that defines a starting or end point of any smooth transition.
    • To animate an element, you first create a named animation, then attach it to an element in
      that elements property declaration block.
    • To create an animation, use the @keyframes rule.
    • Animation Properties: animation-name, ..-duration, ..-timing-function, ..-iteration-count,
      ..-direction, ..-delay, ..-fill-mode, ..-play-state

Chapter 12 Summary - Canvas, SVG, and Drag and Drop

  • Canvas: allows the ability to draw anything useing JavaScript.
    Improves performance by avoiding the need to download images from the network.
  • First step is to add a canvas element to the page.
  • All drawing on the canvas happens via the Canvas JavaScript API.
  • Access canvas functions through canvas.js
  • Use the toDataURL method to save a local copy of a canvas drawing, then right
    click on the image at that URL to save as a PNG or JPEG.
  • SVG = Scalable Vector Graphics - a specific file format that allows you to describe
    vector graphics using XML.
  • SVG's preserve their quality as you blow them up or shrink them down.
  • The Drag and Drop API allows us to specify that certain elements are draggable,
    and then specify what should happen when these draggable elements are dragged
    over or dropped onto other elements on the page.
  • To make an image draggable, add the draggable attribute to the img.
