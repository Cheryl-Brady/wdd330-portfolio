Chapter 10 Summary - Testing and Debugging

  • Errors, exceptions, and warnings:
    • System Error - a problem with the system or external devices with which the
      program is interacting.
    • Programmer Error - the program contains incorrecr syntax or faulty logic (like typos).
    • User Error - user has entered data incorrectly.
    • Exceptions - an error that produces a return value that can be used by the program
      to deal with the error.
    • Stack Trace - a sequence of functions or method calls thaat lead to the point
      where the error occured.
  • Strict Mode: Produces more exceptions and warnings. Better to use to catch errors.
    To enable - add 'use strict'; to the first line of a JS file. *Can also use
    inside a function and it will only be applied to that scope:
    ex: function strictly(){
    'use strict';
    // rest of code
    }
  • Debugging Basics:
    • the alert() method
    • console.log() method
    • console.trace() method - this one will log an interactive stack trace in the console.
  • Debugging Tools:
    • The 'dugger' keyword - this will create a breakpoint wherever you place it.
    • Error objects - created when an exception occurs
    • Throwing exceptions - use the 'throw' stmt
    • Try, Catch, and Finally - Any code placed inside a try block will pass any
      error objects to a catch block when an exception occurs. Any code inside a
      finally block will run if an exception does or does not occur.
  • Test-driven Development (TDD): the process of writing tests BEFORE any actual code.
    1. Write tests (that initially fail)
    2. Write code to pass the tests
    3. Refactor the code
    4. Test refactored code
    5. Write more tests for new features

  • Linting tools can be used to ensure your code follows good practice and conventions.
  • Feature detection can check whether a method is supported before calling it,
    helping to avoid an exception being thrown.
  • The Jest framework can be used to test your code.
