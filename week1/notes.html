JS Reminders:

  • When you want to GET the user INPUT from any input or text area tag, you use the word "value".
    ex: var storyName = document.getElementById("name_imput").value;
  • To SHOW something to a user, you use a section and the word "innerHTML".
    ex: document.getElementById("story_display").innerHTML = storyHTML;

ex: localStorage.setItem(storyName.storyHTML);

** Master Mobile UX - SitePoint Video

*Mobile 1st Design:
"Design for the smallest, least powerful device first... focus on content and core functionality... This leads to simple, beautiful apps."

Things to remember:

  • Desktop computers have been declining since 2011.
  • People are walking around with phones not laptops.
  • Start small and scale up.
  • Mobile is not a fad, its not going away.

Reachability matters:
75% of people use their thumbs to navigate their mobile device. Keep clickable items like menus in thumb reachable places (at the bottom).

Speed matters:

  • Reduce img usage throughout sites/apps.
  • SVG and Webfonts are lower bandwidth.
  • Optimize and minify CSS and JS files.
  • GZIP files to eliminate unnecessary data.

Design Intuitive and Mobile Friendly Forms:

  • Use placeholder text in fields and then float the title.
  • Use number pad pop up for entry in number only fields.
  • Link phone numbers to make direct calls.

Design better advertising:

  • Display smaller ads - make them bigger when clicked.
  • Pop-ups and autoplay videos are despised - stop using them!
  • Keep it simple and sleek, content should come first.
